Project Goal

To build the inhouse product for designers without coding skills for creating customized dashboard for stakeholders running businesses online

Design Approach

Product Design Process (5D)

  • Discover — Validated the requirement through existing tools and stakeholder inputs
    • Competitor Analysis

      Analyzed existing analytics dashboards. Found that most tools are technical and lack real-time flexibility - with few exceptions like Zoho

    • Customer survey for this product

      Created survey questions with the help of GPT-5.3. Used conditional flows to capture more relevant responses based on user context.

    • Research

      Studied how designers and managers currently handle data and dashboards. Found that most rely on manual tracking or pre-built tools

  • Define — Clarified the problem and what the product should solve
    • Persona

      Created two personas - one technical user and one non-technical user. Used this to understand different levels of comfort with APIs and data.

    • Problem Statement

      Designers face difficulty in working with API-based data, especially during mapping and setup. The process needs to be clearer and easier to follow.

    • Focus Areas

      Make the API to dashboard flow simple, clear, and easy to use without deep technical knowledge

  • Design — Worked how users create and manage dashboards
    • Information Architecture

      Structured the flow into clear stages — API import, data mapping, chart selection, and dashboard building

    • Low-Fidelity Wireframes

      Created lo-fi wireframes to test layout and overall flow quickly

    • Interactive Prototype

      Built an interactive prototype to simulate the full workflow and validate usability

  • Develop — Built UI system to support the product
    • Component system

      Designed reusable components for consistent interactions across the product.

    • Design System

      Built a design system in Figma to maintain visual and functional consistency across all screens.

  • Deliver — Finalised designs for implementation
    • Final Screens

      Finalised the UI screens and ensured consistency across the full flow

    • Handoff-ready designs

      Prepared designs in Figma for development and implementation

Challenges
  • Release Handling

    We needed a way to review changes without breaking the main build, so we used separate branches for testing and production

  • Panel Layout → Stepper Change

    Initial design had 3 panels (Data Source, Mapping, Dashboard), which required users to handle multiple steps at once and took more space

    Changed to a step-by-step flow where users add data, select chart type, map data, and then view the result in the dashboard

  • Data Selection Popup Issue

    Data selection was handled in popups, which did not provide enough space to view and work with the data

    Moved to a layout that gives better visibility and control during mapping

  • Mapping & Chart Name Change

    Mapping and chart naming were initially handled together

    Later separated into simpler edit options to improve clarity and due to time constraints

  • Component CSS Issue

    Differences in component styling during development affected consistency and made collaboration between design and development harder

Outcome

A working MVP where designers can import API data, map it to charts, and build dashboards

Clear step-by-step flow for creating and editing charts within the dashboard

Consistent UI ready for development and further improvements
